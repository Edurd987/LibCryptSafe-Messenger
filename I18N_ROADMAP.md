# Локализация (i18n) — дорожная карта

## ЗАЧЕМ
Глобальный инструмент приватности должен работать без языковых
барьеров. Локализация = расширение защищённой сети на новых
пользователей. Делается нативно в Android, логику не трогает.

## ПРЕДУСЛОВИЕ (текущее состояние)
Строки сейчас ЗАХАРДКОЖЕНЫ в коде и layout:
- activity_main.xml: "Чат", "Сеть", "Ещё", "Сообщение..."
- MainActivity.kt: статусы соединения, E2EE, диалоги, диагностика

## ШАГИ (по порядку)
1. РЕФАКТОРИНГ: вынести ВСЕ строки в res/values/strings.xml
   - XML: android:text="@string/tab_chat"
   - Kotlin: getString(R.string.status_connected)
   - проверить что ничего не захардкожено (grep по кавычкам)
2. БАЗОВЫЙ ЯЗЫК: strings.xml = английский (международный дефолт)
3. ПЕРЕВОДЫ: res/values-ru/, values-es/, values-de/ и т.д.
   - НЕ машинный перевод security-терминов (вычитка носителем)
4. RTL (опционально): арабский/иврит — layout уже на start/end ✓

## ВАЖНО
- Чистый рефакторинг ресурсов, НЕ трогает WebSocket/Room/SQLCipher
- Эмодзи-статусы (🔐🟢) универсальны, не локализуются
- Система сама подтягивает язык по локали телефона
