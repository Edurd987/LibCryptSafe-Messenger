# Каналы (вещание новостей) — будущая фича

> Статус: ИДЕЯ, записано. Отдельный крупный кирпич, НЕ часть К5.
> Ценность: в условиях авторитарного режима / блокады информации —
> односторонняя рассылка новостей многим пользователям. Реальная потребность.

## ЧТО ЭТО
Вещание один-ко-многим: автор канала шлёт, подписчики получают (лента).
В отличие от чата (диалог двух peerId) — это broadcast подписчикам.

## ПОЧЕМУ ОТДЕЛЬНЫЙ БОЛЬШОЙ КИРПИЧ (не флаг isChannel)
Каналы ломают текущую крипто-модель:
- ЧАТ: E2E-ключ выводится из ECDH между ДВУМЯ сторонами. Работает.
- КАНАЛ: сообщение для N подписчиков. Как шифровать для многих?
  Нужна ГРУППОВАЯ КРИПТОГРАФИЯ (sender keys, как Signal groups):
  автор шифрует sender-ключом, раздаёт его подписчикам по E2E,
  ротация ключа при изменении списка подписчиков.
- Это отдельный крипто-проект на недели, НЕ "просто флаг".

## АРХИТЕКТУРНЫЕ ВОПРОСЫ (проектировать отдельно)
- как подписчик находит/подписывается на канал (по ID канала?)
- групповой ключ: раздача, ротация, отзыв при отписке
- relay: рассылка подписчикам (fan-out) vs подписчики тянут сами
- анонимность подписчиков (relay не должен видеть список читателей канала?)
- модерация/спам (кто угодно может создать канал?)

## ПОРЯДОК
Делать ТОЛЬКО после того, как базовый мессенджер (чаты, многоконтактность,
офлайн-доставка) работает стабильно. Каналы — надстройка, не фундамент.
Групповая криптография требует отдельного дизайна + аудита.
