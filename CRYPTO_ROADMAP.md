# Крипто-роадмап — очередь крупных задач (приоритеты)

> Записано после того, как К6 упёрся в крипто-ограничение:
> офлайн-доставка невозможна без prekeys. Обходы (QR и пр.) отвергнуты
> как тупиковые — не решают проблему, ломают UX.

## ОЧЕРЕДЬ (строгий порядок по зависимостям)

### 1. X3DH / prekeys — офлайн-доставка (ГЛАВНЫЙ КРИПТО-ПРИОРИТЕТ)
Зачем: доставка тому, кто офлайн / кого не видел давно / после перезапуска app.
Текущее ядро: эфемерный keypair (generateKeypair 1 раз за запуск app),
shared key ломается при перезапуске -> надёжного офлайна НЕТ.
Что нужно (X3DH, как в Signal):
- иерархия ключей: identity key (долгосрочный) + signed prekey +
  пачка one-time prekeys
- публикация prekeys на relay (клиент генерит, подписывает, заливает;
  relay хранит и раздаёт по одному)
- 3-4 DH-операции при установке сессии вместо одного computeSharedKey
- изменения в C++ ядре (CryptoManager): новые external-функции
- хранение prekeys на клиенте и relay (новые таблицы)
- (опционально позже) Double Ratchet поверх — forward secrecy на сообщение
ВАЖНО: самодельная крипта опасна. Развилка (НЕ решена):
готовая аудированная библиотека (libsignal) vs свой с нуля vs сначала дизайн.
Relay-очередь К6 УЖЕ ГОТОВА (SQLite+TTL) — ждёт prekeys.

### 2. АУДИТ на проникновение (Kali Linux) — ПОСЛЕ X3DH
Пентест крипто-ядра через виртуальную Kali.
ПОЧЕМУ ПОСЛЕ: аудировать надо ГОТОВОЕ стабильное ядро, а не черновик.
Ломать до X3DH = ломать то, что всё равно перепишем.
По принципу проекта (дениабельность): "делать только по-настоящему,
с аудитом, иначе вредна" — аудит обязателен после крипто-работы.

### 3. ОНЛАЙН-НАРДЫ (GAME_MOVE через сеть) — независимый довесок
Технически НЕ зависит от X3DH (нарды идут через тот же E2E-конверт К4,
sendGameEvent уже есть). Можно сделать на текущем транспорте.
Приоритет НИЖЕ крипто-ядра (нарды — фича, мессенджер — суть).
Брать, когда захочется переключиться с крипты.

## ПРИНЦИП
Не прыгать между задачами. X3DH -> аудит -> потом довески.
Крипту делать медленно, с проверкой, возможно на аудированной библиотеке.
