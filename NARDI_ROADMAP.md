# Нарды (длинные) — статус и дорожная карта

## ГОТОВО (играбельно, локальный hot-seat)
Полный игровой цикл на одном устройстве (два игрока по очереди):
- доска, шашки, тач-управление, двухэтапный FSM выбора
- зары: бросок, список, дубль = 4 хода, трата заров, отрисовка 1-4 кубиков
- маршрут по кольцу (24 пункта), раздельные пути белых/чёрных, заворот через 0
- легальность хода: направление + дистанция от зара
- ход только своим цветом (turn) + передача хода сопернику
- правило головы (1 шашка с головы за ход)
- запрет вставать на пункт соперника (нет "битья" — длинные нарды)
- сгорание хода при отсутствии легальных ходов
- розыгрыш первого хода (rollOpening: больший зар ходит первым)
- bear-off (выброс из дома) + точный/больший зар
- определение победителя (Toast + блокировка доски)
Доска чистая (debug-номера и логи убраны).

## НЕ СДЕЛАНО — РЕЖИМЫ ИГРЫ (главный якорь!)
Сейчас ТОЛЬКО локальный hot-seat (оба игрока на одном устройстве).
Приложению нужен ЭКРАН ВЫБОРА РЕЖИМА. Три режима:

1. ЛОКАЛЬНО ВДВОЁМ (hot-seat) — ГОТОВО, работает сейчас.

2. ПРОТИВ КОМПЬЮТЕРА (AI-бот) — ГОТОВ (примитивный жадный).
   ВНИМАНИЕ: сейчас бот ЖЁСТКО включён в GameActivity (botEnabled=true).
   Hot-seat временно недоступен. Нужен экран выбора режима.
   Усиление бота (эвристики/минимакс) — на будущее.
   [старое описание ниже]
   - нужен движок принятия решений: выбор хода из легальных
   - порядок: сначала ПРИМИТИВНЫЙ бот (случайный/жадный легальный ход),
     потом усиление (эвристики/минимакс)
   - НЕ требует сети — можно делать сразу
   - использует уже готовую hasAnyLegalMove / isLegalMove / canBearOff

3. ОНЛАЙН ПО СЕТИ — НЕ СДЕЛАНО, ЗАБЛОКИРОВАНО транспортом.
   - нужен протокол синхронизации: GAME_MOVE через E2EE
   - (в мессенджере тип GAME_MOVE заложен, но для нард не подключён)
   - БЛОКЕР: wss/Cloudflare транспорт не дотестирован end-to-end
   - делать ТОЛЬКО после рабочего транспорта

### Рекомендованный порядок реализации режимов:
1) примитивный бот (быстро, без сети) -> режим "с компьютером"
2) рабочий транспорт (wss/Cloudflare) -> разблокирует сеть
3) сетевой режим (GAME_MOVE по E2EE)
4) экран выбора режима (когда есть из чего выбирать)
